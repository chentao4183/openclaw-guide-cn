# OpenClaw 安装指导手册

> **面向用户**： 技术小白、非技术用户、刚刚接触OpenClaw的开发者
> **目标**： 让任何人在5分钟内快速部署一个可用的OpenClaw实例

---

## 🎯 核心理

**简单至上**： 不搞复杂概念，只讲实操
**降低门槛**: 让小白也能轻松上手
**快速验证**: 安装完立即能看到效果

---

## 📋 壴分钟快速部署流程

### 方案一：极速体验（最简单，推荐)

**适合人群**: 想快速尝鲜、不确定是否长期使用

**步骤**:
1. **安装OpenClaw CLI**
   ```bash
   # Linux/macOS
   curl -fsSL https://openclaw.ai/install.sh | bash
   
   # Windows (PowerShell)
   iwr -useb https://openclaw.ai/install.ps1 | iex
   ```

2. **运行新手向导**
   ```bash
   openclaw onboard
   ```
   - 选择"快速开始"模式（推荐默认值)
   - 设置模型提供商（推荐Anthropic API密钥）
   - 工作区使用默认路径
   - 皂箱模式使用默认设置（安全)
   - 皂箱模式使用默认设置
   - Gateway端口: 18789
   - Gateway认证: 自动生成token
   
3. **选择渠道（推荐Telegram最简单）**
   - **Telegram** (最快)
     1. 创建Bot: 在BotFather与创建机器人
     2. 获取Token: 复制token
     3. 配置: 在向导中粘贴token
   4. 启动网关: `openclaw gateway`
   
   - **WhatsApp** (需二维码)
     1. 运行: `openclaw channels login`
     2. 扫码: 用WhatsApp扫描二维码
     3. 配置: 向导会自动写入配置
   
   - **Discod** (简单)
     1. 创建Bot: 在Discord Developer Portal创建
     2. 获取Token: 复制token
     3. 配置: 在向导中粘贴token
     4. 启动网关: `openclaw gateway`
   
   - **WebChat** (无需配置,最快验证)
     1. 启动Dashboard: `openclaw dashboard`
     2. 访问: http://127.0.0.1:18789/
     3. 聊天: 在浏览器中直接聊天
   
4. **验证安装**
   ```bash
   openclaw status
   openclaw health
   ```
   看到"Gateway is running"和健康状态信息
   
5. **发送测试消息**
   ```bash
   # 替换为你的电话号码
   openclaw message send --target +1234567890 --message "Hello"
   ```

## 🚀 方案二:标准部署(生产环境,推荐)

**适合人群**: 部署到服务器、长期使用、需要稳定服务

**步骤**:
1. **安装OpenClaw CLI**
   ```bash
   # Linux/macOS
   curl -fsSL https://openclaw.ai/install.sh | bash
   
   # Windows (PowerShell)
   iwr -useb https://openclaw.ai/install.ps1 | iex
   ```

2. **运行新手向导**
   ```bash
   openclaw onboard
   ```
   - 选择"高级"模式
   - 设置认证方式（推荐API密钥）
   - 配置Gateway:
     * 端口: 自定义（如8080）
     * 绑定: 0.0.0.0或特定IP
     * 认证: Token（推荐）或关闭
     * Tailscale: 可选
   - 配置渠道（选择需要的渠道）
   - 配置守护进程:
     * Linux: systemd
     * macOS: launchd
   
3. **启动服务**
   ```bash
   openclaw gateway start
   ```
   
4. **验证**
   ```bash
   openclaw status
   openclaw health
   ```

5. **配置自动更新** (可选)
   ```bash
   # 设置定时检查
   openclaw cron add --name health-check \
     --schedule '{"kind": "every", "everyMs": 3600000}' \
     --payload '{"kind": "agentTurn", "message": "检查系统健康状态"}'
   ```

## 🐧 方案三:容器化部署
**适合人群**: 熟悉Docker的用户、需要隔离环境

**步骤**:
1. **安装Docker**
   - 繁体中文用户: 安装Docker Desktop
   - Linux用户: 安装Docker Engine

2. **拉取OpenClaw镜像**
   ```bash
   docker pull openclaw/openclaw:latest
   ```

3. **运行容器**
   ```bash
   docker run -d \
     --name openclaw \
     -p 18789:18789 \
     -v ~/.openclaw:/root/.openclaw \
     openclaw/openclaw:latest
   ```

4. **进入容器配置**
   ```bash
   docker exec -it openclaw openclaw onboard
   ```
   在容器内完成配置

5. **访问Dashboard**
   http://localhost:18789

## 🔧 常见问题排查

### 问题1: 找不到openclaw命令
**症状**: 运行`openclaw`提示"command not found"

**原因**: PATH未包含npm全局bin目录

**解决方案**:
```bash
# 检查npm全局路径
npm prefix -g
echo $PATH

# 添加到PATH (macOS/Linux)
echo 'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# Windows PowerShell
$env:PATH += ";$(npm prefix -g);"
```

---

### 问题2: Gateway启动失败
**症状**: `openclaw gateway`报错"Port already in use"

**原因**: 端口18789被占用
**解决方案**:
```bash
# 查找占用进程
lsof -i :18789  # macOS/Linux
netstat -ano | findstr : :18789  # Windows

netstat -ano | findstr : :18789

# 停止占用进程或更换端口
kill -9 <PID>
# 或者
openclaw gateway --port 18790
```

---

### 问题3: WhatsApp无法登录
**症状**: `openclaw channels login`失败

**原因**: 网络问题或二维码过期
**解决方案**:
1. 检查网络连接
2. 重新运行登录命令
3. 使用备用手机号码
4. 检查防火墙设置
```

---

### 问题4: 认证配置错误
**症状**: Agent无响应或提示"authentication not configured"

**原因**: 未设置API密钥或OAuth配置错误
**解决方案**:
```bash
# 重新运行向导
openclaw configure

# 或手动设置API密钥
export ANTHROPIC_API_KEY="your-key"
openclaw onboard --auth-choice apiKey --anthropic-api-key "$ANTHROPIC_API_KEY"
```

---

### 问题5: 权限问题
**症状**: Gateway无法访问工作区或配置文件
**原因**: 文件权限不正确
**解决方案**:
```bash
# 检查权限
ls -la ~/.openclaw

# 修复权限
chmod -R u+rw ~/.openclaw/workspace
```
