# 渠道配置

OpenClaw 支持多个聊天平台，选择你想配置的渠道：

| 渠道 | 难度 | 说明 |
|------|------|------|
| [WhatsApp](./whatsapp.md) | ⭐⭐ | 最受欢迎，扫码登录 |
| [Telegram](./telegram.md) | ⭐ | 需创建 Bot，配置简单 |
| [Discord](./discord.md) | ⭐⭐ | 需创建 Bot 应用 |

---

## 访问控制策略

在配置渠道前，了解访问控制很重要：

### DM（私信）策略 - `dmPolicy`

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| `pairing` | 未知用户需配对码批准（默认） | 安全，适合个人使用 |
| `allowlist` | 只允许白名单用户 | 严格限制 |
| `open` | 允许所有人 | 公开服务 |
| `disabled` | 禁止所有私信 | 只用群组 |

### 群组策略 - `groupPolicy`

| 策略 | 说明 |
|------|------|
| `allowlist` | 只允许白名单群组 |
| `open` | 允许所有群组 |
| `disabled` | 禁止群组消息 |

### 触发方式

默认情况下，群组中需要 `@提及` 机器人才会响应。

---

## 配置文件位置

- **主配置文件**: `~/.openclaw/openclaw.json`
- **格式**: JSON5（支持注释和尾随逗号）
- **热重载**: 大部分配置修改后自动生效
