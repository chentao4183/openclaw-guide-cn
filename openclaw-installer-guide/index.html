🎯 OpenClaw 快速部署指南

5分钟内让AI助手在你的聊天应用中运行

适合人群

想快速尝鲜，不确定是否长期使用

步骤

# 1. 安装OpenClaw CLI 
# Linux/macOS
curl -fsSL https://openclaw.ai/install.sh | bash

# Windows (PowerShell)
iwr -useb https://openclaw.ai/install.ps1 | iex
# 2. 运行新手向导 
openclaw onboard

openclaw health
# 4. 打开Dashboard 
openclaw dashboard
✅ 完成! 浏览器会自动打开 http://127.0.0.1:18789

适合人群

部署到服务器、长期使用、需要稳定服务

步骤

# 1. 安装OpenClaw CLI 
# Linux/macOS
curl -fsSL https://openclaw.ai/install.sh | bash

# Windows (PowerShell)
iwr -useb https://openclaw.ai/install.ps1 | iex
# 2. 运行新手向导（高级模式） 
openclaw onboard
⚙️ 配置选项:
  • 认证方式: API密钥或OAuth
  • Gateway端口: 默认18789
  • 渠道: Telegram/WhatsApp/Discord
  • 守护进程: systemd/launchd
# 3. 启动服务 
openclaw gateway start
# 4. 验证 
openclaw status
openclaw health

适合人群

熟悉Docker的用户、需要隔离环境

步骤

# 1. 拉取OpenClaw镜像 
docker pull openclaw/openclaw:latest
# 2. 运行容器 
docker run -d \
  --name openclaw \
  -p 18789:18789 \
  -v ~/.openclaw:/root/.openclaw \
  openclaw/openclaw:latest
# 3. 进入容器配置 
docker exec -it openclaw openclaw onboard
# 4. 访问Dashboard 
# 浏览器访问 http://localhost:18789

❓ 找不到openclaw命令?

原因: PATH未包含npm全局bin目录
解决方案: 
# 检查npm全局路径
npm prefix -g
echo $PATH

# 添加到PATH (macOS/Linux)
echo 'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

❓ Gateway启动失败?

症状: `openclaw gateway`报错"port already in use"
解决方案: 
# 查找占用进程
lsof -i :18789  # macOS/Linux
netstat -ano | findstr : :18789  # Windows

netstat -ano | findstr : :18789

# 停止占用进程或更换端口
kill -9 
# 或者
openclaw gateway --port 18790

❓ WhatsApp无法登录?

症状: `openclaw channels login`失败
解决方案: 
1. 检查网络连接
2. 重新运行登录命令
3. 使用备用手机号码
4. 检查防火墙设置
